{$in_}Crear archivo{$_in}

El archivo debe estar dentro de una carpeta dentro del proyecto.
Los nombres de capeta se recomienda que se organicen en dos niveles sucesivos (in<interfaz>/<tecnología>) seguido por un tercer nivel que representa el nombre del proyecto, o su area específico de aplicación:
  • Internacionalizado (prefijo: in) + Interfaz de usuario que utilizan:
    • innui: Internacionalicado + No interactua con el usuario (usa cadenas de texto, datos, archivos, etc. sin intervención del usuario)
    • Ejemplos del nivel siguiente:
      • archivo: trabaja con archivos
      • ente: estructuras de datos: entidades
      • html: trabaja con formatos de presentación HTML
      • json: trabaja con formatos de datos JSON
      • webtec: html + plantillas
      • xml: trabaja con formatos de datos XML
    • incli: Internacionalicado + cliente de servidor (No interactúa con el usuario y se comunica con servidores)
    • Ejemplos del nivel siguiente:
      • database: trabaja con clientes de bases de datos
      • restful: trabaja con arquitectura RestFull
      • soap: trabaja con arquitectura SOAP
      • tcp: trabaja con arquitectura TCP
      • web: trabaja con arquitectura web
    • inser: Internacionalicado + servidor (No interactúa con el usuario y se comunica con servidores)
    • Ejemplos del nivel siguiente:
      • igual que en: incli
    • inclui: Internacionalicado + Interfaz de linea de comando (combina la entrada-salida estandar de consola)
    • Ejemplos del nivel siguiente:
      • stream: Lee de la entrada estandar, y escribe en la salida estandar, como una corriente de datos
      • chat: Lee datos, y los escribe, tras interacciones intermitentes con el usuario
    • ingui: Internacionalicado + Interfaz grafico (interactúa con eventos gráficos del usuario)
    • Ejemplos del nivel siguiente:
      • javafx: trabaja con la tecnología Java FX
      • swing: trabaja con la tecnología Swing
      • gtk: trabaja con librerías gráficas GTK
      • wpf: trabaja con librerías gráficas Windows
